# lintree-py

[![Версия](https://img.shields.io/badge/version-1.2.0-blue.svg)](CHANGELOG-ru.md)

`lintree-py.py` - это однофайловый TUI-визуализатор использования дискового пространства для Python 3.8. Он сканирует дерево каталогов, считает занимаемое место и показывает интерактивную цветную treemap-карту прямо в терминале.

Реализация использует только стандартную библиотеку Python.

## Возможности

- Одновременный сканер файловой системы с ограниченным пулом потоков.
- Исключение каталогов по имени или пути, а также исключение NFS/CIFS/SMB mount points.
- Горизонтальная wide-раскладка по умолчанию и исходная square-раскладка.
- Интерактивное переключение метрики отображения: по размеру или по количеству файлов.
- Цветовая кодировка типов файлов: код, web-файлы, документы, изображения, видео, аудио, архивы, данные, бинарные файлы, каталоги и прочее.
- Изменение яркости по активной метрике, чтобы крупные или насыщенные файлами элементы сразу выделялись.
- Псевдографические границы treemap и более понятная рамка выбранной ячейки.
- Интерактивный вход в каталоги и возврат назад.
- Хлебные крошки с текущим расположением.
- Боковая панель с именем, типом, активным режимом, размером, счетчиками файлов, путем, процентом от родителя и крупнейшими дочерними элементами.
- Индикатор режима сканирования на экране прогресса и в статусной строке.
- Поддержка клавиатуры и выбора мышью в терминалах с mouse reporting.
- Работа на Linux, macOS и Windows через ANSI-последовательности и ввод из стандартной библиотеки.

## Требования

- Python 3.8 или новее.
- Терминал с поддержкой ANSI-цветов. Лучше всего выглядит в терминалах с truecolor.

Сторонние пакеты не нужны.

## Использование

```bash
python3.8 lintree-py.py
python3.8 lintree-py.py /var/log
python3.8 lintree-py.py --fast ~/Downloads
python3.8 lintree-py.py --layout square /var/log
python3.8 lintree-py.py --layout wide /var/log
python3.8 lintree-py.py --exclude-dir node_modules --exclude-dir .git .
python3.8 lintree-py.py --exclude-network-fs /
```

## Параметры

```text
path                  Каталог для сканирования. По умолчанию текущий каталог.
--fast                Использовать больше потоков сканирования. `-fast` оставлен как alias для совместимости.
--layout wide         Предпочитать широкие горизонтальные области. Это режим по умолчанию.
--layout square       Использовать исходную squarified-раскладку.
--horizontal          Алиас для --layout wide.
--square              Алиас для --layout square.
--exclude-dir VALUE   Пропустить каталоги по имени или пути. Можно повторять или перечислять через запятую.
--exclude-network-fs  Пропустить каталоги на NFS, CIFS или SMB файловых системах.
--exclude-fs-type T   Пропустить mount points по типу ФС. Можно повторять или перечислять через запятую.
-v, --version         Показать версию.
-h, --help            Показать справку.
```

## Управление

| Клавиша | Действие |
|---|---|
| `↑` `↓` `←` `→` | Пространственный переход к соседней ячейке |
| `j` / `k` | Переход по ячейкам в визуальном порядке чтения |
| `Enter` / `l` | Открыть выбранный каталог |
| `Backspace` / `h` | Вернуться назад |
| `o` | Переключить wide/square-раскладку |
| `m` | Переключить отображение по размеру/файлам |
| `?` | Показать или скрыть справку |
| `q` / `Ctrl+C` | Выйти |
| Клик мышью | Выбрать ячейку |

## Режимы раскладки

Режим `wide` оптимизирован для чтения в терминале. Крупные элементы становятся широкими горизонтальными областями, а мелкие группируются в нижние строки. Это помогает читать подписи на очень широких терминалах.

Режим `square` сохраняет исходное поведение squarified treemap, где алгоритм стремится строить компактные прямоугольники с более сбалансированными пропорциями.

## Метрики отображения

Нажмите `m`, чтобы переключить treemap между режимами `size` и `files`.

В режиме `size` площадь прямоугольников, яркость ячеек, подписи, `% Parent` и списки дочерних элементов считаются по занимаемому месту. В режиме `files` те же элементы интерфейса считаются по количеству файлов, поэтому каталоги с большим числом мелких файлов становятся заметны даже при небольшом размере.

## Архитектура

Проект специально оставлен в одном файле:

- `ConcurrentScanner` строит дерево `FileNode` с помощью рабочих потоков.
- `compute_sizes_and_sort` считает агрегированные размеры каталогов и количество элементов.
- `node_metric` выбирает вес для отрисовки: размер или количество файлов.
- `layout_treemap` преобразует дочерние элементы в прямоугольники treemap с учетом активной метрики отображения.
- `Canvas` собирает кадр в памяти и выводит ANSI truecolor.
- `TerminalController` управляет raw-режимом терминала и вводом с клавиатуры/мыши.
- `LintreeApp` связывает сканирование, навигацию, раскладку и отрисовку.

Диаграммы PlantUML находятся в каталоге `diagramms/`.

## Примечания

- Символические ссылки пропускаются, чтобы избежать циклов.
- `--exclude-dir` принимает простое имя вроде `node_modules` или путь вроде `/mnt/cache`.
- `--exclude-network-fs` пропускает mount points NFS/NFS4, CIFS, SMBFS, SMB3 и fuse.smbnetfs до сканирования.
- `--exclude-fs-type` позволяет исключить любой тип смонтированной ФС, видимый в таблице mount.
- На Unix размер предпочитает занятые блоки (`st_blocks * 512`), если эти данные доступны.
- На Windows используется логический размер файла, если данные о занятых блоках недоступны.
- `/proc`, `/sys`, `/dev` и `/run` пропускаются на Unix при сканировании root-подобных путей.
